# OpenTSC v1.0 安装使用升级迁移命令

本文档是 OpenTSC Skill v1.0 (Soul/Shell 架构) 的命令速查表。

Skill 目录：

```text
I:\dsd\opentsc\.claude\skills\opentsc
```

CLI 入口：

```text
I:\dsd\opentsc\.claude\skills\opentsc\scripts\opentsc.py
```

以下命令默认在项目根目录执行，默认 vault 路径为 `opentsc`。
如果你的 vault 不叫 `opentsc`，把命令里的 `--root opentsc` 改成你的 vault 路径。

---

## 0. 查看全部命令

```bash
python .claude/skills/opentsc/scripts/opentsc.py --help
```

---

## 1. 首次安装 / 初始化

首次使用运行：

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc init
```

v1.0 会创建完整的 soul/shell/world 目录结构：

- `soul/` — 魂（_genesis.md, _judgment_codex.md, _rule_codex.md, _schema.md, events/, calibration/）
- `shell/` — 壳（kernel/, modules/_registry.md, professions/11个VSM职业, genesis_engine/）
- `world/` — 世界模型（players/, npcs/humans/, npcs/agents/, orgs/, operations/, roles/）
- 旧版目录同时保留（people/, orgs/, operations/ 等）以兼容旧命令
- ledger 账本、inbox 候选、knowledge 知识、actions 行动等

## 3. v0.4 → v1.0 迁移

**新增命令**。将旧版扁平 vault 迁移到 soul/shell/world 架构：

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc migrate
```

迁移会：

- 创建 soul/shell/world/ 目录
- 移动 `people/` → `world/npcs/humans/`（默认为人类 NPC）
- 移动 `orgs/` → `world/orgs/`
- 移动 `operations/` → `world/operations/`
- 提取内嵌事件 → `soul/events/` 独立节点
- 从 `_doctrine.md` 生成 `_genesis.md` + `_rule_codex.md` + `_judgment_codex.md` 种子
- 移动 `feedback/` → `soul/calibration/`
- 初始化 11 个 VSM 职业
- 升级人物格式为三层属性 (base/skills/states)
- **保留所有原始数据**，写迁移清单到 `ledger/migration_v04_v10.json`

迁移后所有旧命令继续工作。

## 6. 事件图谱 (新增 — 核心升级)

### 添加事件（自动触发 K7 判断引擎）

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc event-add B2 "Carol 按时完成报价文档交付" "会议纪要" --link p_carol --link op_eve
```

添加事件后，K7 判断引擎会自动：
1. 匹配 judgment_codex 中的触发词
2. 更新相关实体的 base 属性 / 技能等级 / buff-debuff
3. 留推理痕迹在事件的 judgment_triggered 字段

### 链接事件到更多实体

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc event-link evt_20260605_5b1c076b p_alice o_acme
```

### 建立因果关系

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc event-cause evt_20260601_abc evt_20260602_def
```

### 读取单条事件

### 查询事件时间线

## 7. 判断引擎 K7 (新增 — 核心创新)

### 查看实体属性

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-attribute p_carol reliability
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-attribute p_carol execution_ceiling
```

返回 AttributeClaim：{value, confidence, provenance, reviewed, decay}

### 直接比较两人

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-compare p_carol p_dave negotiation
```

不重读历史，直接从已推导的属性值回答"谁更强"。

### 解释属性来源

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-explain p_carol reliability
```

返回推导链：哪些事件、什么证据、什么逻辑。

### 衰减过时属性

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-decay
```

长时间没有新证据的属性，confidence 自动衰减。

### 清理过期 buff/debuff

```bash
python .claude/skills/opentsc/scripts/opentsc.py --root opentsc judgment-clean-states
```
