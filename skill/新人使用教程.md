# OpenTSC Skill 新人使用教程

这份文档给第一次使用 OpenTSC 的人看。

你不需要会写命令，不需要懂文件夹，不需要知道 Python，也不需要手动改模板。

你只需要会做一件事：

> 把你知道的人、事、项目，用自然语言告诉 AI。AI 会按照 OpenTSC 的规则帮你建档、整理、追踪和复盘。

OpenTSC 是一个 **Skill**。它不是让你亲自操作数据库，而是让 Claude、Codex 或其他 agent 在需要时自动调用这套能力。

---

## 1. OpenTSC 到底是干什么的

OpenTSC 是一个私人情报系统。

它帮助你记住：

- 你认识了谁
- 这个人是谁介绍来的
- 这个人做过什么事
- 哪些事说明他靠谱
- 哪些事说明他有风险
- 哪些项目正在推进
- 哪些判断以后要用结果验证

它不是通讯录，也不是 CRM，也不是员工表。

更准确地说：

> OpenTSC 是把你脑子里对人和项目的判断，变成可以查询、可以复盘、可以校准的系统。

---

## 2. 你应该怎么使用 OpenTSC Skill

你不用打开模板填表。

你只需要对 AI 说类似这样的话：

```text
帮我把张三录入 OpenTSC。
他是 Carol 介绍来的，做技术，最近参与杭州培训项目。
我还不确定他靠不靠谱，先标记为待观察。
```

或者：

```text
帮我记录一条 OpenTSC 事件：
张三今天按时提交了 API 测试报告，内容完整，阿波罗确认可以用于客户交付。
来源是杭州项目群聊天记录。
关联人物：张三、阿波罗。
关联项目：杭州陈总培训项目。
```

AI 会负责：

- 判断这是新增人物、事件、项目，还是关系
- 给人物分配稳定 ID
- 把自动提取的内容放入草稿或正式记录
- 保留来源和证据
- 让 OpenTSC 的判断引擎根据事件更新人物画像

你不用直接碰底层文件。

---

## 3. 你和 OpenTSC 的分工

### 你负责提供真实材料

你告诉 AI：

- 谁
- 发生了什么
- 什么时候发生
- 来源是什么
- 你确定还是不确定
- 这件事和哪些人、哪些项目有关

### OpenTSC Skill 负责整理

AI 会帮你：

- 建人物档案
- 建项目档案
- 记录事件
- 关联人物和项目
- 生成待确认草稿
- 提醒哪些信息缺失
- 根据事件推导可靠性、执行力、技能、风险状态

### 系统负责长期判断

OpenTSC 不要求你直接说“这个人靠谱”或“这个人不靠谱”。

更好的方式是记录事件：

```text
张三连续两次按时交付。
张三在关键节点已读不回。
张三主动提前说明自己做不了。
张三提供了可验证的项目材料。
```

系统会根据这些事件慢慢形成判断。

---

## 4. 新增一个人时，你应该怎么说

最简单的说法：

```text
帮我把张三录入 OpenTSC。
他是 Carol 介绍来的，微信名叫小李，做技术，参与杭州项目。
目前我只知道这些，其他先留空。
```

更完整的说法：

```text
帮我把张三录入 OpenTSC。

基本信息：
- 姓名：张三
- 常用称呼：小李
- 微信名：ZhangSan_Dev
- 城市：杭州
- 介绍人：Carol
- 认识来源：Carol 在项目群里介绍
- 当前关联项目：杭州陈总培训项目
- 已知能力：API 测试、技术文档、后端接口排查
- 已知风险：周末回复不稳定
- 当前状态：待观察

请先建立人物档案，不要直接给他高信任评分。
```

这句话里最重要的是最后一句：

> 不要直接给他高信任评分。

因为 OpenTSC 的原则是：**属性从事件推导，不靠手填。**

---

## 5. 记录事件时，你应该怎么说

事件是 OpenTSC 最重要的材料。

你可以这样说：

```text
帮我记录一条 OpenTSC 事件。

时间：2026-06-05
内容：张三按时提交了 API 测试报告，报告内容完整，阿波罗确认可以用于客户交付。
来源：杭州项目群聊天记录
可信度：比较可信，但还没独立复核
关联人物：张三、阿波罗
关联项目：杭州陈总培训项目
```

AI 会把它整理成 OpenTSC 能使用的事件。

---

## 6. 什么是好事件

好事件像这样：

```text
张三在 2026-06-05 按时提交了 API 测试报告，报告内容完整，阿波罗确认可用于客户交付。
```

它有：

- 谁
- 什么时间
- 做了什么
- 结果是什么
- 来源是什么

不好的事件像这样：

```text
张三很靠谱。
```

这不是事件，这是结论。

OpenTSC 可以保存结论，但结论必须有证据。更好的方式是把证据记录进去，让系统自己推导。

## 7. 你可以用这些句式

### 新增人物

```text
帮我把某某录入 OpenTSC。
我知道的信息如下：……
不确定的地方请帮我列成待确认。
```

### 补充人物信息

```text
帮我更新某某的 OpenTSC 信息。
他新增了一个微信名：……
他现在参与的项目是：……
这些信息来源于：……
```

### 记录正面事件

```text
帮我记录一条正面事件。
某某在某项目里做了什么，结果是什么，来源是什么。
```

### 记录风险事件

```text
帮我记录一条风险事件。
某某在关键节点出现了什么问题，影响是什么，来源是什么。
先作为待确认，不要一次性定性。
```

### 查询一个人

```text
帮我查一下 OpenTSC 里某某目前的画像。
重点看可靠性、执行力、已知技能、当前风险状态。
```

### 比较两个人

```text
帮我比较某某和某某，谁更适合负责这个任务。
请基于 OpenTSC 里的事件，不要只凭印象。
```

### 复盘一个项目

```text
帮我复盘某项目。
请列出关键事件、参与人表现、风险点、后续行动建议。
```

---

## 8. 你不用说得很专业

你可以直接说：

```text
今天张三还行，按时把东西交了。
```

但更好一点是：

```text
今天张三在杭州项目里按时交了 API 测试报告，阿波罗看过说可以交给客户。
```

你不需要懂 OpenTSC 的字段。

你只要尽量把事情讲清楚。

AI 会追问缺失信息，比如：

- 这件事发生在哪一天？
- 来源是什么？
- 关联哪个项目？
- 这条信息是你亲眼看到，还是别人转述？
- 是否需要先放入草稿，等你确认？

---

## 9. 新人信息从哪里来

新增一个人时，信息通常来自三种地方：

1. 你自己知道
2. 介绍人告诉你
3. 新人自己填写问卷

其中最推荐的是：

> 先让新人填写问卷，再把问卷交给 OpenTSC Skill 处理。

问卷不是为了审问新人，而是为了减少合作误会。

---

## 10. TSC 成员问卷

下面这份问卷可以直接发给 TSC 成员填写。

你可以这样对成员说：

```text
这是 TSC 的成员信息问卷，用来帮助团队了解你适合参与什么项目、怎么跟你协作最顺。
不方便填写的地方可以写“不方便”，不确定的地方可以写“不确定”。
真实比漂亮重要。
```

# TSC 成员信息问卷

## A. 基本信息

1. 真实姓名：

2. 希望团队怎么称呼你：

3. 英文名、昵称、网名：

4. 微信名：

5. Telegram 用户名：

6. 手机号，可选：

7. 邮箱，可选：

8. 所在城市：

9. 常用工作时间：

   例：工作日 10:00-19:00；晚上可回复；周末不稳定。

10. 不方便被打扰的时间：

---

## B. 你是怎么进入 TSC 的

1. 谁介绍你认识 TSC（本组织）？

2. 你和介绍人是什么关系？

   例：朋友、同事、客户、合作伙伴、群友、以前项目认识。

3. 你们认识多久了？

4. 介绍人对你的评价是什么？不知道可以写“不知道”。

5. 你是因为什么事情接触 TSC 的？

---

## C. 你现在能做什么

请用普通话写，不需要专业术语。

1. 你最擅长的三件事是什么？

   - 第一件：
   - 第二件：
   - 第三件：

2. 你做过哪些真实项目？请写 1 到 5 个。

   每个项目可以按这个格式写：

   ```text
   项目名称：
   你负责什么：
   最后结果：
   有无可验证材料：
   ```

3. 你会哪些工具、软件、平台？

   例：Python、剪映、飞书、Telegram、Midjourney、Excel、Notion、WordPress。

4. 你会哪些语言？

   例：中文、英文、日语、粤语。

5. 你能独立完成什么类型的任务？

6. 哪些任务你可以配合别人完成，但不能独立负责？

7. 哪些任务你明确不擅长？

---

## D. 你希望承担什么角色

1. 你更喜欢哪类工作？可以多选。

   - 执行具体任务
   - 对外沟通
   - 客户谈判
   - 写方案
   - 做研究
   - 做技术
   - 做设计
   - 做运营
   - 做销售
   - 做项目管理
   - 做复盘总结
   - 其他：

2. 你不喜欢哪类工作？

3. 你希望自己在 TSC 里变成什么样的人？

4. 你最希望从 TSC 得到什么？

   例：赚钱、项目经验、人脉、能力提升、长期事业机会。

---

## E. 可靠性与工作习惯

这一部分不是考试，是为了减少误会。

1. 你通常多久回复消息？

   例：马上、1小时内、半天内、一天内、不固定。

2. 如果你忙不过来，会不会主动说？

3. 如果你做不了一件事，会不会提前讲？

4. 你最容易在哪些情况下拖延？

5. 过去有没有因为拖延、失联、误解导致项目出问题？如果有，请简单说一次。

6. 你希望别人怎么跟你协作，你效率最高？

7. 你最不喜欢别人怎么跟你协作？

---

## F. 压力与边界

1. 你能接受临时加急任务吗？

2. 你能接受晚上或周末处理紧急事情吗？

3. 你不能接受的工作边界是什么？

   例：违法灰产、不清楚权责、长期无偿、频繁熬夜、没有明确交付标准。

4. 你在压力大的时候通常是什么状态？

   例：沉默、急躁、效率变高、效率变低、需要别人帮忙拆任务。

5. 如果你快扛不住了，你会怎么提醒团队？

---

## G. 资源与可调动能力

1. 你身边有没有可以合作的人？比如技术、设计、销售、渠道、客户资源。

2. 这些资源是你能直接调动，还是只能介绍认识？

3. 你有没有客户资源或行业资源？

4. 你熟悉哪些行业？

5. 你有没有已经做过或正在做的个人项目？

---

## H. 成本与合作方式

1. 你希望按什么方式合作？

   - 按项目收费
   - 按天收费
   - 按月合作
   - 按结果分成
   - 先低成本试合作
   - 其他：

2. 你的大概报价或期望收入是多少？

3. 如果是试合作，你能接受什么样的试合作方式？

4. 你最看重合作里的什么？

   例：钱、成长、长期机会、信任、自由度、资源。

## 11. 问卷填完后，怎么交给 OpenTSC Skill

你不需要自己整理字段。

把问卷内容直接发给 AI，然后说：

```text
这是张三填写的 TSC 成员问卷。
请你用 OpenTSC Skill 处理：
1. 提取人物基本信息
2. 建立或更新人物档案
3. 把能力、边界、合作方式整理成草稿
4. 把可验证材料作为证据事件
5. 不确定的信息标记为待确认
6. 不要直接给高信任评分
```

如果你希望更稳一点，可以加一句：

```text
先不要正式入册，先生成待确认草稿给我审阅。
```

这符合 OpenTSC 的原则：

> 自动提取先进入草稿，确认后再正式入册。

---

## 12. AI 处理问卷时应该输出什么

你可以要求 AI 输出下面四块：

```text
1. 人物档案草稿
   - 建议 ID
   - 姓名和别名
   - 介绍人
   - 标签
   - 当前关联项目

2. 证据事件草稿
   - 他声称自己做过什么
   - 有哪些材料能证明
   - 来源是什么
   - 可信度如何

3. 风险与边界
   - 回复速度
   - 不适合的任务
   - 压力状态
   - 合作禁区

4. 待确认问题
   - 哪些信息缺失
   - 哪些能力没有证据
   - 哪些说法需要后续项目验证
```

这样你审起来会很快。

## 13. 一次完整示例

你发给 AI：

```text
帮我用 OpenTSC 录入一个新人。

问卷摘要：
姓名：张三
称呼：小李
介绍人：Carol
城市：杭州
能力：API 测试、技术文档、接口排查
做过项目：某 SaaS 系统接口测试，提供过测试报告
回复习惯：工作日 1 小时内，周末不稳定
合作方式：按项目收费
边界：不接长期无偿、不接违法灰产、不适合周末应急

请先生成 OpenTSC 草稿，不要直接高信任入册。
```

AI 应该返回类似：

```text
已识别为新人录入任务。

建议人物 ID：p_zhangsan
建议标签：技术、杭州、待观察
介绍关系：p_carol -> p_zhangsan，状态待确认

建议事件草稿：
1. 张三通过 Carol 介绍接触 TSC，表达参与技术类任务意愿。
2. 张三自述具备 API 测试、技术文档、接口排查能力。
3. 张三说明周末回复不稳定，适合工作日任务，不适合周末高压应急。

待确认：
- API 测试报告是否可查看
- Carol 对张三的真实评价
- 张三是否已参与过 TSC 内部项目
```

你确认后，AI 再正式写入 OpenTSC。

## 14. 日常使用口诀

不用记命令，记这几句话就够了：

```text
帮我把这个人录入 OpenTSC。
帮我记录一条事件。
帮我更新这个人的信息。
帮我查一下这个人的画像。
帮我比较这两个人谁更适合这个任务。
帮我复盘这个项目。
```

OpenTSC Skill 的价值，不是让人学会操作工具。

它的价值是：

> 让人正常说话，系统在背后把人际情报整理成可用的结构。
