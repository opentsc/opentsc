# 许可与商业授权 / Licensing

OpenTSC 采用**双许可（dual-license）**模式。

## 1. 社区/开源许可：AGPL-3.0

默认情况下，OpenTSC 的代码以 **GNU Affero General Public License v3.0**（见 [LICENSE](LICENSE)）发布。这意味着你可以自由使用、修改、分发，但：

- 任何分发的修改版本必须以 AGPL-3.0 开源；
- **即使你只是把它作为网络服务提供给用户**（不分发二进制），也必须向这些用户提供完整源码。

对个人、研究、开源项目，AGPL-3.0 完全够用且免费。

## 2. 商业许可（闭源 / 免 AGPL 约束）

如果你希望：

- 把 OpenTSC 集成进**闭源**产品；
- 作为 SaaS 提供但**不想开源**你的修改；
- 获得无 copyleft 约束的使用权、保修或支持；

则需要向版权方购买**商业许可**。商业许可解除 AGPL 的开源回馈义务。

**联系获取商业授权**：dashen · [dashen.wang](https://dashen.wang) · [@dashen_wang](https://x.com/dashen_wang)

## 3. 为什么能这样做

OpenTSC 的版权由创始人 dashen 持有。所有外部贡献者在提交代码前需签署 [贡献者许可协议（CLA）](CONTRIBUTING.md)，把贡献的版权许可授予项目，从而保证版权方有权同时以 AGPL 和商业许可两种方式分发。这是 dual-license 开源项目（如 MySQL、Qt、GitLab）的标准做法。

## 4. 白皮书另行许可

理念/白皮书（[opentsc/tsc](https://github.com/opentsc/tsc)）以 **CC BY-SA 4.0** 单独发布，与本仓代码许可相互独立。
